Abstract

Data analytics is essential for extracting valuable insights from data that can assist organizations in making effective decisions. We introduce InsightBench, a benchmark dataset with three key features.

  1. InsightBench Introduction. InsightBench consists of 31 datasets representing diverse business use cases such as finance and incident management, each accompanied by a carefully curated set of insights planted in the datasets.
  2. End-to-End Analytics Evaluation. Unlike existing benchmarks focusing on answering single queries, InsightBench evaluates agents based on their ability to perform end-to-end data analytics, including formulating questions, interpreting answers, and generating a summary of insights and actionable steps.
  3. Quality Assurance. We conducted comprehensive quality assurance to ensure that each dataset in the benchmark had clear goals and included relevant and meaningful questions and analysis.
  4. Two-Way Evaluation Mechanism. We implement a two-way evaluation mechanism using LLaMA-3-Eval as an effective, open-source evaluator method to assess agentsâ€™ ability to extract insights.
  5. Baseline Agent: AgentPoirot. We propose AgentPoirot, our baseline data analysis agent capable of performing end-to-end data analytics. Our evaluation on InsightBench shows that AgentPoirot outperforms existing approaches that focus on resolving single queries.
  6. Comparison and Access. We compare the performance of open and closed-source LLMs and various evaluation strategies. Overall, this benchmark serves as a testbed to motivate further development in comprehensive data analytics and can be accessed "https://github.com/ServiceNow/insight-bench".

AgentPoirot

We propose AgentPoirot, which, given a dataset (D) in InsightBench and a goal (G), performs systematic data exploration aimed at achieving G. The working of AgentPoirot is as follows:

  1. Extract Dataset Schema (S): AgentPoirot extracts information about every column in the dataset, including name, type, total unique values, total NA values, and additional statistics such as minimum, maximum, mean, and standard deviation for numerical data, and minimum and maximum dates for date data. For other data types, it extracts the top 5 unique values from each column.
  2. Generate Insights: Given the triplet (D, G, S), AgentPoirot uses the Question Generation Prompt to generate k (equal to 3) high-level questions. It answers each high-level question by generating Python code and interpreting the output. Then, it generates n (equal to 4) follow-up questions for each high-level question and answers them as well, resulting in a total of (n+1)Ã—k insights. (see Figure 2 to see a complete overview of AgentPoirot)
  3. Generate Summary: AgentPoirot summarizes the insights obtained from the previous step using Prompt 7.

Additionally:

  • generic goal: We use the generic goal, 'I want to find interesting insights in this dataset,' instead of a carefully designed goal and run AgentPoirot on InsightBench.
  • non-diverse follow-ups: We use Prompt 4 instead of Prompt 3 to generate only one type of follow-up question and run AgentPoirot on InsightBench.
